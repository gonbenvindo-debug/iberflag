# Arquitetura da Oferta

## 1. Objetivo

Organizar a oferta para que:

- o cliente compre mais depressa;
- a equipa venda com mais consistencia;
- a margem nao seja destruida por excesso de customizacao nao controlada.

## 2. Estrutura-base da oferta

### Core offer

Produtos que devem carregar a maior parte da procura:

- fly banners
- bandeiras
- roll up
- photocall
- wall banner
- tendas

Estes produtos precisam de:

- pagina forte;
- preco de entrada claro;
- variantes claras;
- processo de personalizacao simples;
- prazo e expectativa visiveis.

### Expansion offer

Produtos complementares ou menos frequentes:

- mini fly banners
- balcoes
- cubos publicitarios
- mastros
- mochilas fly banner

Objetivo:

- aumentar ticket medio;
- servir necessidades mais especificas;
- funcionar como upsell ou cross-sell.

### Service layer

Servicos que podem transformar margem:

- revisao tecnica de arte
- adaptacao de ficheiro
- urgencia de producao
- instalacao ou apoio especial, se fizer sentido mais tarde
- reposicao / reimpressao facilitada

## 3. Modelo de oferta recomendado

Para cada familia de produto, usar 3 niveis simples:

### Nivel 1: Essencial

- opcao de entrada
- menos decisoes
- menor preco
- adequada para quem quer rapidez

### Nivel 2: Recomendado

- melhor relacao entre estabilidade, visibilidade e custo
- deve ser o caminho preferido na maioria das categorias

### Nivel 3: Profissional

- maior tamanho, robustez, acabamento ou impacto visual
- indicado para uso intensivo, feiras, exterior ou marcas mais exigentes

Isto reduz choice overload e melhora comparacao.

## 4. Como apresentar variantes

Evitar:

- lista longa de nomes tecnicos sem contexto;
- mostrar todas as configuracoes ao mesmo tempo;
- obrigar o cliente a perceber sozinho a diferenca entre bases, tecidos e tamanhos.

Fazer:

- mostrar `ideal para interior`, `ideal para exterior`, `mais facil de transportar`, `mais estavel`
- destacar uma opcao `mais escolhida`
- limitar defaults a uma opcao sensata

## 5. Add-ons que fazem sentido

### Add-ons de valor real

- base premium
- saco de transporte
- dupla face
- montagem mais rapida
- revisao de ficheiro
- envio expresso

### Add-ons a tratar com cuidado

- elementos pouco percebidos pelo cliente
- extras que criam atrito antes do pagamento
- opcoes tecnicas que devem estar escondidas ate serem relevantes

## 6. Bundles recomendados

Criar pacotes por caso de uso em vez de vender so itens isolados.

Exemplos:

- `Kit Feira Pequena`
  - 1 photocall
  - 2 fly banners
  - 1 balcao

- `Kit Loja`
  - 1 roll up
  - 1 bandeira exterior

- `Kit Evento Exterior`
  - 1 tenda
  - 2 flags
  - pesos / bases adequadas

Beneficios:

- aumenta ticket medio;
- simplifica decisao;
- posiciona a IberFlag como consultiva, nao apenas catalogo.

## 7. O que entra no preco base

Isto tem de estar claro.

Decidir e comunicar:

- se o preco inclui impressao;
- se inclui estrutura;
- se inclui saco;
- se inclui uma revisao de ficheiro;
- se inclui IVA ou se o site mostra sem IVA, dependendo da estrategia comercial.

Ambiguidade aqui mata conversao e margem.

## 8. O que exige validação humana

Nem tudo deve ser completamente automatico.

Fluxos que podem exigir controlo:

- artwork complexo;
- grandes quantidades;
- urgencia acima do SLA normal;
- NIF / VAT intracomunitario;
- pedidos com risco fiscal ou logistico;
- produtos ainda com configuracao incompleta.

## 9. Heuristicas comerciais por categoria

### Fly banner

- `mais visivel`
- `mais escolhido para eventos`
- `mais facil de montar`

### Photocall

- `ideal para stands e fundos de palco`
- `montagem rapida`
- `grande area de impacto visual`

### Roll up

- `mais pratico para interior`
- `transporte simples`
- `solucao economica para ponto de venda`

### Tendas

- `maior presenca de marca`
- `ideal para exterior`
- `estrutura de maior valor`

## 10. Regras de simplificacao

1. Cada pagina deve orientar para uma decisao, nao para dez.
2. Cada categoria precisa de um recomendado.
3. Cada add-on tem de provar valor.
4. O cliente nao deve ter de aprender linguagem de producao para comprar.

## 11. Backlog imediato

1. Definir os produtos core que merecem investimento prioritario.
2. Criar recomendados por categoria.
3. Definir bundles por caso de uso.
4. Distinguir claramente o que e base, opcional e premium.
5. Rever paginas para remover opcoes e labels tecnicas desnecessarias.
